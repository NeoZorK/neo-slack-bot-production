# Project Structure - Build and Organization

## Overview

This document covers header organization, build artifacts, and code structure standards.

## Header Organization

### Standard Header Structure

```cpp
#ifndef MODULE_NAME_HPP
#define MODULE_NAME_HPP

// System includes
#include <string>
#include <vector>

// Project includes
#include "other_module.hpp"

// Forward declarations
class SomeClass;

// Class definition
class ModuleName {
    // ...
};

#endif // MODULE_NAME_HPP
```

### Include Order

1. Corresponding header (for `.cpp` files)
2. System headers (`<...>`)
3. Project headers (`"..."`)
4. Forward declarations

### Implementation Organization

**Standard `.cpp` Structure**:
```cpp
#include "module_name.hpp"

// Implementation details
namespace {
    // Internal helpers
}

// Class implementation
// Methods in logical order
```

## Build Artifacts

### Generated Directories

**`src/obj/`**: Object files from compilation
- `.o` files for each source file
- Generated by compiler
- Excluded from version control

**`src/bin/`**: Executable files
- `neo-slack-bot` - Main application
- Generated by linker
- Excluded from version control

**`tests/obj/`**: Test object files
- Similar to `src/obj/`
- For test compilation

**`tests/bin/`**: Test executables
- `test_neo_slack_bot` - Test runner
- Generated by linker

### Cleanup

Remove all artifacts created by CMake:
```bash
cd /path/to/neo-slack-bot
rm -rf build
```
